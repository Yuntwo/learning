本地模型代理 + 抓包观测
把 Claude Code 的模型调用链路看清楚：转发与抓包分层配置，不再混淆端口职责。
推荐链路
Claude Code → 抓包代理 :9090 → 本地模型代理 :8787 → 目标模型服务 抓包端口负责观察，请求转发端口负责处理，不需要改成同一个端口。
1
先过抓包，再进模型代理
只要流量顺序正确，就能同时保留本地兼容入口和完整请求观测能力。
2
别把 127.0.0.1 放进 NO_PROXY
否则本地 8787 会直接绕过抓包代理，表现为调用成功但抓不到任何包。
3
HTTPS 证书要一起处理
想看明文请求与响应，除了配代理，还要信任证书，必要时补 NODE_EXTRA_CA_CERTS。
4
调试大模型链路特别高效
适合定位请求是否发出、是否命中本地代理，以及输入输出与报错细节。
