## 核心观点

当多模态大模型的视频理解能力已经足够强时，落地的关键不再是"让模型学会"（能力获取），而是"让模型按要求执行"（能力激活与任务适配）。因此工程重心应从"训练/微调"转向"工程编排"——通过上下文工程、提示词工程、输出 schema 约束与校验，在不改模型参数的前提下，把通用能力适配到垂直场景。

***

## 要点整理

### 1. 技术选型：为什么不微调，而走工程链路

对比三种路径：

* **人工标注**：成本高、覆盖率低、标签体系一变就要重训标注团队，响应慢。

* **模型训练（微调专用模型）**：每次标签体系变更都要重新走"采集→标注→训练→评估"全流程，迭代重、周期长。

* **工程链路（采用）**：基模已具备视频-文本对齐的泛化理解力；任务本质是"给定视频→按维度输出结构化结果"，正是大模型擅长的；新增/修改维度只需改任务描述与约束，迭代成本极低。

### 2. 应用架构：单 Prompt → Workflow → 单 Agent → 多 Agent

本质是从"单次推理"走向"工程化编排"，再走向"自主决策""协同决策"。\
当业务动线固定、无需模型自主决策时，**Workflow（工作流）架构比 Agent 更合适**——主要工作在编排各节点逻辑与状态机控制。

### 3. 分层设计：内容理解层 与 下游子任务层 分离

| 原则    | 好处            | 体现                                                           |
| ----- | ------------- | ------------------------------------------------------------ |
| 单一职责  | Prompt 更精准    | 每个 Prompt 只做一件事（理解 or 生成），注意力集中                              |
| 可组合复用 | 新场景冷启动快、成本可优化 | 视频理解最耗资源，理解结果作为结构化中间产物传给下游，一个视频不重复理解；新场景复用理解产物，只写新子任务 Prompt |
| 独立迭代  | 调优成本低         | 每次只调一个环节，评测可分层验证                                             |

### 4. 落地场景（内部业务，⚠️脱敏点）

* **AI 标签自动生产**：从预定义标签体系识别内容维度标签（情节映射 / 设定 / 时代背景 / 受众 / 身份设定），实现小时级自动打标落库。〔内部收益数据已略，不外发〕

* **智能倍速**：对每集按"剧情变化优先"切片，对片段做高光打分（10 分制），高光慢放、平淡加速，实现"精彩多看、无聊快进"。〔内部收益数据已略，不外发〕

### 5. 工程中的典型问题与解法（通用，价值最高）

| 问题                                           | 解法                                                               |
| -------------------------------------------- | ---------------------------------------------------------------- |
| 角色识别跨集不一致（全剧上下文超窗口）                          | 集间串行传递角色信息，手动构造跨轮次长期记忆，建剧维度角色库（含外貌/身份特征）                         |
| 边界标签误判（语义相近、embedding 表征距离近，如"都市逆袭" vs "马甲"） | 在提示词显式注入判定条件 = 给模型一条**判别式推理链（discriminative CoT）**，把隐式分类转成显式条件逻辑 |
| 输出格式不稳定（JSON 字段缺失/越界）                        | 严格 schema 校验 + 重试/修正                                             |
| 标签体系动态扩展                                     | 标签体系与输出示例结构都用传参方式动态拉取，不动 Prompt 主体                               |
| 切片精度不够（抽帧低帧率 ±2s 误差、切在台词/bgm 中间）             | 提高抽帧 fps（0.5→1）；引入 ASR 句子边界 + AED 音频段落边界作为约束，切点对齐自然断点            |

### 6. 方法论提炼

**上下文工程三原则**

1. **补齐模型信息盲区**：模型知识 = 训练数据 + 当前输入。偏差源于信息缺失时，优先补输入而非改指令（例：角色库注入）。

2. **多模态互补优于单模态精调**：单模态信息天花板确定，瓶颈来自模态局限时，引入互补模态投产比 > 反复调 Prompt（例：ASR+AED 时间戳注入）。

3. **采样密度与 Token 预算权衡**：fps 越高精度越高但 Token 线性增长，按任务对精度的敏感度选最优采样率（标签任务低采样、高光任务 fps=1）。

**提示词工程三原则**

1. **明确定义优于模糊描述**：业务术语与通用语义有偏差时，必须显式给判定条件，把分类转成条件逻辑判断。

2. **用示例定义输出格式**：Few-Shot 模型会模仿最近见过的输出模式；陷阱——示例里出现标签体系外的枚举值（如"甜宠逆袭"），模型会当合法选项复现。

3. **用 CoT 引导逐步推理**：多步判断任务（先判类型→再判情节→定标签）要求模型先输出中间推理步骤再给结论，避免跳步。

**输出校验与修正**

* Schema 强校验（字段完整性、类型、业务规则）。

* 校验不过→把"错误信息 + 原始输出 + 修正指令"构造为新输入触发自修正。

* 本质是把 Reflection 架构（生成→评估→反馈修正）工程化，用代码校验替代人工审核。

### 7. 未来方向

* 切片时间精度细化：引入关键帧/优选帧提取作为硬约束；ASR 精确时间戳 + OCR 字幕 + 语义分句 agent。

* Token 成本优化：分级输入（已采用）、自适应采样率（按镜头切换密度动态调 fps）、关键帧去重。

* 多特征提取与短剧资产建设。
