Claude Code 的多轮上下文到底是怎么带的？

不是服务端神秘记忆，而更像是：每次请求前，客户端重新组织 system、messages、多轮历史、工具轨迹与必要摘要。

关键点 01
system 放全局规则与环境，messages 放多轮历史
关键点 02
assistant 负责 tool_use，tool_result 常作为 user 侧回灌
关键点 03
上下文太长时，旧历史可能 summarize，而不是永远原样保留
一句话结论
多轮“记忆”本质上是客户端持续重建上下文
适合收藏的 Agent 机制笔记 Context ≈ 重放 + 压缩
