协议学习笔记

A2A 协议到底是什么？

它不是替代 HTTP，而是构建在 HTTP 之上的 Agent 协作协议：用 JSON-RPC 描述动作，用 task 语义管理一次长期任务。

一眼看懂

  • 底层通常还是 HTTP(S)
  • 动作常写在 JSON-RPC 的 method 里
  • message/send 更像方法名，不是 REST path
  • 流式更新常通过 SSE 承载

核心对象

Agent Card / Message / Task / Artifact

从“请求响应”升级到“任务协作”。

和 REST 的区别

REST 更看 path + verb，A2A 更看统一入口 + JSON-RPC method + task 生命周期。

常见方法

message/send、message/stream、tasks/get、tasks/cancel、tasks/resubscribe。

为什么适合多 Agent

能表达能力发现、任务跟踪、流式更新和断线续订，不用每个 Agent 都自定义私有协议。

学习方式

最有效的是配着 demo 看：先看 Agent Card，再用 curl 发一个 message/send，就能立刻建立直觉。

关键词：HTTP / JSON-RPC / SSE / Agent Card / Task
