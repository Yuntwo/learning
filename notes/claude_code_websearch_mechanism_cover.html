WebSearch
到底是谁在输出？
一篇讲清楚模型 SSE、tool_use 和 CLI 本地渲染分层的工程笔记
核心结论
模型输出的是结构化 tool_use 事件；终端看到的 Web Search("...") 和耗时，是 CLI 本地渲染出来的执行日志。
1
SSE 里只有 tool_use + JSON 参数
raw 协议层看到的是 content_block_start / delta / stop，不是自然语言日志。
2
CLI 会把工具调用 prettify
WebSearch 会被渲染成 Web Search，并抽出 query 生成更易读的终端展示。
3
Did 1 search in 1s 不是模型 token
这是工具执行完成后，本地 harness 才知道的统计反馈。
4
决策和执行是分层的
模型负责提出工具调用，runtime 负责执行工具，再把结果回注给模型。
