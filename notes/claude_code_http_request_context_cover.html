LLM 工程实践
一次抓包看懂
Claude Code 怎么喂模型
从真实请求结构出发，拆清 prompt、messages、context 和流式输出的关系
核心结论
agent 产品发给模型的，从来不是“用户一句话”，而是一整包被编排过的运行时上下文。
1
prompt 不是单一字段
system、messages、tools description 都在共同影响模型行为。
2
tools 也是上下文负载
工具不仅是 schema，更是告诉模型何时调用、如何调用的行为说明。
3
用户输入常常只占很小一部分
大头 token 往往来自系统规则、环境说明、能力清单和缓存前缀。
4
流式输出本质是 SSE 事件流
真实客户端要按事件累计，而不是把第一段响应当成完整结果。
