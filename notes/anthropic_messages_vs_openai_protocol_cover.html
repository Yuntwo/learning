协议对比速查

Anthropic Messages vs OpenAI

两者都在解决大模型对话、工具调用和流式输出，但 path、system 位置、tool 编排和 streaming 事件模型都不一样。
对比项 Anthropic Messages OpenAI Chat / Responses 速记
常见 path /v1/messages /v1/chat/completions
/v1/responses		 先看 URL，常能先判断协议家族
system 位置 顶层 system Chat 放 messages.system
Responses 放 instructions		 切协议时，先重组 system prompt
content 结构 更偏 block 数组 更常见字符串、parts 或 delta 复杂场景别只用纯字符串抽象
tool 调用 tool_use / tool_result tool_calls / role: tool 字段不同，消息编排也不同
参数形态 结构化 input 对象 常见 JSON 字符串 arguments 别忘了 parse 和校验 schema
停止原因 stop_reason finish_reason 监控和日志口径要统一
token 统计 input_tokens / output_tokens prompt_tokens / completion_tokens 计费层要做字段映射
streaming 风格 message / block 生命周期事件 delta 增量片段或 event stream 前端解析别写死单一供应商格式

一句话结论

Anthropic 和 OpenAI 是同类接口，但不是同协议。做兼容，不能只改 model 或 URL。

工程建议

  • 内部先统一抽象：systemPrompt、contentBlocks、toolCalls、usage、streamEvents。
  • path、消息体、streaming 三层分别做适配。
适合大模型网关、SDK 封装、Agent 框架、多模型兼容场景
