把牛客 SQL 快速入门 42 题通关后的踩坑整理成一份速查卡，给要面试/做数据的同学一份"少走弯路"清单👇

1️⃣ 执行顺序 ≠ 书写顺序
FROM → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT
记住这条顺序，后面所有坑都能自洽解释。

2️⃣ 别名作用域
WHERE 里用 SELECT 的别名会报错；但 GROUP BY / HAVING / ORDER BY 在 MySQL、PostgreSQL、Hive 里都能用。ORDER BY 跨库都能用，因为它在 SELECT 之后才执行。

3️⃣ SUM(CASE…) 忘写 ELSE 0 = 返回 NULL
最经典的坑：算正确率时，某分组一条 right 都没有，SUM(全 NULL) 返回 NULL 而非 0，最终得到 None。
修法：加 ELSE 0；或者直接换 COUNT(CASE…) — COUNT 对全 NULL 返回 0。
➡️ COUNT(CASE WHEN result='right' THEN 1 END) / COUNT(*)

4️⃣ LEFT JOIN 的 ON vs WHERE
从表条件写 WHERE 会把"左表没匹配到"的行过滤掉，LEFT JOIN 就退化成 INNER JOIN。
要保留左表全集 → 从表条件必须放 ON。

5️⃣ 每组 TOP-N = 窗口函数
"每个学校 GPA 最低的同学"这种需求，一行搞定：
ROW_NUMBER() OVER (PARTITION BY university ORDER BY gpa ASC)
= 1 即可取出每组第 1 名；想并列都留用 RANK() / DENSE_RANK()。

6️⃣ 列被函数包住 = 走不了索引
WHERE YEAR(dt)=2021 用不到 dt 上的索引，改写成：
WHERE dt >= '2021-01-01' AND dt < '2022-01-01'

额外小坑：
· NOT IN 列表里有 NULL 就恒空集
· INT 除法可能截断，保险写法乘 1.0
· LIMIT m,n 顺序容易记反，推荐 LIMIT n OFFSET m
· CASE 别名写在 END 之后，写进内部会语法错

把这份放桌面当速查表，面试/工作少被坑 80%。
